Dina Kalendar – Supabase i Vercel

Korak po korak: baza, prijava i besplatno hostovanje.

1. Supabase – baza i prijava

1 Otvori supabase.com i napravi besplatan nalog (Sign in / Sign up).
2 Klikni New project. Izaberi organizaciju (ili napravi novu), unesi ime projekta i lozinku za bazu – sačuvaj je negde.
3 Kad se projekat učita, u levom meniju idi na Project Settings (zupčanik), pa API. Tu ćeš naći:
  • Project URL – to je vrednost za NEXT_PUBLIC_SUPABASE_URL
  • anon public (ključ) – to je NEXT_PUBLIC_SUPABASE_ANON_KEY
Kopiraj oba negde (npr. u Notepad) – trebaće ti za lokalno pokretanje i za Vercel.
4 U levom meniju otvori SQL Editor (ikona </>). Tu ćeš pokrenuti tri SQL skripte jednu za drugom (vidi sekciju 2 ispod).
(Opciono) Ako želiš da korisnici mogu da se uloguju odmah bez potvrde emaila: Authentication → Providers → Email → isključi Confirm email.

2. SQL migracije u Supabase-u

U SQL Editor u Supabase-u radi sledeće. Za svaku skriptu: + New query → nalepi ceo sadržaj fajla → Run.

1 Otvori u projektu fajl supabase/migrations/001_initial.sql. Kopiraj ceo sadržaj (Ctrl+A, Ctrl+C), nalepi u SQL Editor, klikni Run. Treba da piše da je uspešno.
2 Novi query: otvori supabase/migrations/002_predavanja_i_ucenik.sql, kopiraj ceo sadržaj, nalepi, Run.
3 Novi query: otvori supabase/migrations/003_instructor_color.sql, kopiraj ceo sadržaj, nalepi, Run.
Provera: u levom meniju otvori Table Editor. Trebalo bi da vidiš tabele: instructors, clients, terms, predavanja.

3. Lokalno pokretanje

1 U folderu projekta (Dina kalendar) napravi fajl .env.local. U njemu stavi (zameni vrednosti onim iz Supabase koraka 3):
NEXT_PUBLIC_SUPABASE_URL=https://tvoj-projekat.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=tvoj_anon_key_ovde
2 U terminalu (u tom istom folderu) pokreni:
npm install npm run dev
3 Otvori u browseru http://localhost:3000. Klikni „Registrujte se” i napravi prvog predavača.

4. Vercel – hostovanje na internetu

1 Otvori vercel.com i uloguj se (najlakše preko GitHub-a).
2 Klikni Add New → Project. Izaberi repozitorijum sa ovim projektom (npr. MudroUcenjeDina). Ako ga nema na listi, prvo poveži GitHub nalog u Vercel podešavanjima.
3 Pre nego što klikneš Deploy, otvori Environment Variables. Dodaj dve promenljive (iste vrednosti kao u .env.local):
  • Name: NEXT_PUBLIC_SUPABASE_URL   Value: tvoj Supabase Project URL
  • Name: NEXT_PUBLIC_SUPABASE_ANON_KEY   Value: tvoj Supabase anon key
4 Klikni Deploy. Kad se završi, Vercel će ti dati link tipa https://mudro-ucenje-dina.vercel.app (ili slično). Sačuvaj taj URL.

5. Podešavanje Supabase-a za produkciju

Da bi prijava radila na tom Vercel linku, u Supabase-u moraš dozvoliti taj domen.

1 U Supabase-u: Authentication → URL Configuration.
2 Site URL postavi na tvoj Vercel URL, npr. https://mudro-ucenje-dina.vercel.app.
3 U Redirect URLs dodaj (jedan red): https://tvoj-vercel-url.vercel.app/auth/callback (zameni sa svojim pravim URL-om). Klikni Save.

Gotovo. Aplikacija je na besplatnom domenu (Vercel), baza i prijava na Supabase-u.
